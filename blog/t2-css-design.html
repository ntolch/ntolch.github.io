Absolutely Relative

July 20, 2015

Dear Past Self...I'm about explain what'll be the cause of quite a bit of head-scraching so completely that you'll wish I'd left something of the subject for you to research and discover on your own.

The position property lets you control an element's type of positioning on the webpage. There are 4 values you can give it: static, relative, fixed, and absolute. The static position is the default value--the element will simply do what it would've otherwise done. So we won't worry about that.

The relative position affects where the element will be positioned in relation to it normal, or static, position. Initially, I kept thinking of this value as making elements move around other around. You know, in relation to the other objects. An element with a fixed position is positioned relative to the browser. It stays in exactly the same place on the screen when you scroll through the webpage. Other elements ignore an object that is in a fixed position when figuring out where to place themselves. Here's an example:

The absolute position is one of my favorites and, funny enough, is the most complicated. It position an element relative to the closest parent element that has relative or absolute positioning. It's similar to the fixed position in that it makes an element sort of invisible to other elements--the element isn't affected by the other objects and vice versa. Except the absolute position fixes the element to the nearest positioned parent instead of to the screen. Time for an example!
