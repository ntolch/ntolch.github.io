What's Your Function?

August 29, 2015

Dear past self: I'm gonna teach you about functional programming. It sounds scary, I know. That's because it is. Well, at first. But remember, we fear what we don't understand. It'll get better, I promise.

There are two basic concepts you have to know when dealing with functional programs:

  1. Data must be immutable (unchanging). Instead of changing existing data, you just create a new data structure with the updated data.
  2. Programs should be without ‘memory’ of the past. So each time the program runs should be like the first time. Feel like the very first time.

Still sound scary? Shhh shhh, it's ok... After the initial learning pains, it won't hurt you. It's there to help! How, you ask? And is it worth it? Well...good questions. It can make your programs easier to understand and faster. Instead of looping (like going through a list of items and doing the same thing to each), functional programming is more like making a list of items where a function is being used on each. Ah...ok, that just confused both you and me more. Um...ok, usually, fucntional programming involves using other functions to help define the main function--sometimes even using the main function inside itself. (What!) It's like breaking up chunks of your code that work towards a similar task and giving them their own name.

Also, functional programming doesn't have side effects. Take this, for example: 

        function () {
          console.log("Hello, world!")
        };

That print statement? That there's a side effect. Let's take a look at code without any. 

        function hello () {
          greet = console.log("Hello, world!")
          return greet
        };

Now that there's a functional program! You see? Well...maybe not. Ok, so in second program, we're describing what we want done rather than executing or carrying it out. Just keep practicing, past self. This helped, but practice is the only way you'll really get it.
