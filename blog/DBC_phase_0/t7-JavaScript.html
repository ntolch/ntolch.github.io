Objects vs Hashes

August 23, 2015

JavaScript's objects are very similar to ruby's hash objects. The differences can be subtle.

First, let's look at how you access a value from each. Both JavaScript objects and ruby hashes contain key/value pairs, like so: secretAgent = {codeName: "Pink Panter"} In JavaScript, you would access the value of codeName by typing secretAgent.codeName to get Pink Panter. In ruby, however, you'd get the same value by typing secretAgent[:codeName]. (I told you the differences were subtle.)

The difference in creating new key/value pairs is much the same. In JavaScript, you could add to our secretAgengt object with this: secretAgent.secretWeapon = "exploding lighter";. In ruby, you'd do this: secretAgent[:secretWeapon] = "ticking timebomb pocketwatch".

Also, JavaScript objects are variables within a variable. Ruby hashes contain instance variables. (Did I mention how subtle the differences could be?) This difference mainly has to do with scope. In JavaScript, there are essentially only 2 levels of scope--in a function and outside a function. In ruby, however, there are a few more levels of scope. Instance variables are accessed only by an instance of the class in which the hash is created.
